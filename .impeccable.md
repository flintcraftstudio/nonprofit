# Flintcraft Studio — Design Context

This is the **advanced-tier starter template** for Flintcraft Studio. Pages shipped
here are the default that cloned client/app projects start from and adapt.

## Design Context

### Users
Two audiences, one codebase:
- **Public landing** — visitors evaluating an advanced-tier app/product. It is
  **login-forward**: the primary action leads to sign-in, with feature highlights
  that communicate the product is fast, secure, and well-built.
- **Protected `/admin`** — the authenticated operator (a known admin, created via
  the seed CLI). Their job is to manage the app. The default admin page is a
  **welcoming shell** that teaches where project features plug in.

### Brand Personality
**Handcrafted, quiet, precise.** Confidence through restraint, not noise. The
interface should feel *built* — like forged metalwork — not rendered. Three words:
forged, calm, deliberate. Emotional goal: a craftsman's quiet competence.

### Aesthetic Direction — "Flint & Ember"
- **Theme:** dark, warm. Cool **flint charcoal** neutrals (a desaturated warm-gray
  stone ramp) lit by a single warm **ember** spark accent. The tension between cool
  stone and warm spark is the whole identity — never introduce a third hue.
- **Type:** editorial serif moments in **Cormorant Garamond** (display) over
  **DM Sans** (body). Let the serif carry headlines and large index numerals.
- **Motion:** drifting embers/sparks in the background canvas; otherwise minimal.
  One orchestrated page-load is worth more than scattered micro-interactions.
  Always honor `prefers-reduced-motion`.
- **Anti-references:** generic SaaS card-grids; neon-on-black; purple→blue gradients;
  gradient text; glassmorphism; identical icon-topped feature cards; centered-
  everything layouts. If it looks like default AI output, it's wrong.

### Color Tokens (`ff-*`, defined in tailwind/tailwind.config.js)
- **Flint neutrals:** `dark` (page) → `dark2` → `panel` → `panel2` (surfaces),
  warmed near-black to warm charcoal.
- **Light text:** `paper` (brightest, warm white) → `cream` → `moon` → `ash`
  (muted secondary) → `stone` (faint). Tinted warm, never pure white.
- **Ember accent:** `ember` (primary) / `ember-hover` / `ember-mid` (muted, borders) /
  `ember-lo` (deep wash, glow cores). `glow` is the warm-gold light accent for links
  and focus rings.
- **Primary buttons** use **dark ink text on ember** (not white) — more legible and
  more distinctive than white-on-orange.

### Design Principles
1. **Cool stone, warm spark.** Vast calm charcoal space; ember used sparingly as a
   true accent. If ember is everywhere, it stops being a spark.
2. **Serif carries the voice.** Headlines and large numerals in Cormorant; body and
   labels in DM Sans. Editorial, not corporate.
3. **Asymmetry over grids.** Left-aligned, varied spacing, hairline rhythm. Avoid
   identical card rows; size things by importance.
4. **Empty states teach.** The admin shell shows operators *where* features go, not
   "nothing here."
5. **Quiet motion.** Drifting embers and one page-load reveal; respect reduced motion.
